# PRD: DT-C2 单元格内联编辑机制

## 1. 需求背景
为提升用户录入效率，DataTable 需支持在当前页面直接修改单元格内容，而无需跳转到独立的编辑页面。

## 2. 功能描述
* **双态切换**: 单元格默认为只读文本模式，双击后原位切换为编辑控件。
* **状态隔离与防抖**: 编辑中产生的高频 State 更新保持在小组件内部逻辑层。仅在用户确认或失焦后，再触发 Yjs 同步更新。
* **异常处理**: 支持使用 `Esc` 键放弃当前修改，恢复原始状态。

## 3. 验收标准
| ID | 描述 | 优先级 | 验证方式 |
|---|---|---|---|
| AC-1.1 | 交互响应：双击单元格后 100ms 内完成到 Edit 状态的切换。 | P0 | 单元测试/手动验证 |
| AC-1.2 | 局部渲染：格内输入时，父级 Table 组件不应发生重绘动作。 | P0 | Profiler 工具探测 |
| AC-2.1 | 确认保存：触发 `onBlur` 且内容变化时，需自动调用 Yjs 原生提交接口。 | P0 | 网络 Patch 抓包 |
| AC-2.2 | 放弃机制：在编辑状态下点击 `Esc` 需立即回退至只读状态而不保存变更。 | P1 | 功能交互测试 |

## 4. 技术实现架构
```mermaid
graph LR
    A[只读状态 <span>] -->|双击| B(编辑状态 <input> 自动聚焦)
    B -->|高频内容变更| C(拦截在 <ReactiveCell /> 内部状态中缓冲)
    C --> B
    B -->|确认/失焦| D{数据是否有实质变动?}
    D -->|是| E[调用 original.set() 同步至 Yjs 并持久化] --> A
    D -->|否/取消| A
```

## 5. UI/UX 细节
* **高亮效果**: 编辑状态下的单元格应增加特定的边框投影作为聚焦标识。
* **Portal 挂载**: 针对部分 Select 下拉选择器，需使用 Portal 挂载到 body 顶层，防止被表格父容器的 `overflow: hidden` 截断遮挡。
