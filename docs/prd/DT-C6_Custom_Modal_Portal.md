# PRD: DT-C6 复杂逻辑自定义 Modal 插槽

## 1. 需求背景
部分业务字段涉及极其复杂的计算逻辑，无法简单的通过单行输入框完成。组件需支持宿主挂载自定义的复杂弹窗组件。

## 2. 功能描述
* **Render 插槽**: 暴露 `cellEditor` 回调接口，参数包含当前行数据及回传钩子。
* **Portal 脱离容器**: 内部机制支持通过 React Portal 将宿主 Modal 渲染到 body 顶层，绕过表格的溢出限制（overflow）。
* **同步节流**: 弹窗操作期间不进行中间态网络同步，仅在最终调用 Commit 钩子时执行单笔状态更新。

## 3. 验收标准
| ID | 描述 | 优先级 | 验证方式 |
|---|---|---|---|
| AC-1.1 | 插槽能力：配置了自定义 Editor 的列双击后需正确打开宿主注入的 React 组件内容。 | P0 | 组件对接测试 |
| AC-2.1 | 网络隔离：Modal 打开及内部多步表单填报期间，后台抓包保持 0 字节协同流量。 | P0 | 通信监控 |
| AC-2.2 | 完成保存：弹窗点击保存并调用 commit 后，对应单元格数据需刷新且在远端屏幕同步成功。 | P0 | 端到端保存链条测试 |

## 4. 技术逻辑
```mermaid
graph TD
    Trigger[双击开启特殊列单元格] --> Check{配置了自定义 cellEditor?}
    Check -->|是| P[创建渲染 Portal 加载宿主提供的 Modal 实例]
    P --> |本地独立交互期| IDLE[网络协同保持静默]
    IDLE --> |触发 commit(payload)| WR[回传数据注入组件内部]
    WR --> O[执行 Yjs 最终覆写操作并触发网络同步]
```

## 5. 注意事项
* **交互体验**: 由于组件将 Modal 弹起至顶层 Body，宿主需自行处理 Z-index 的层叠竞争及弹窗在窗口缩放时的自适应对齐策略。
